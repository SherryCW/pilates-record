# Pilates Record｜普拉提训练记录

一个前端优先的普拉提训练记录工具。用户可以浏览动作图片、按器械分类或关键词搜索动作，选择当天的练习，填写重量、次数和组数，并在完成后生成可下载的训练分享图。

## 这个 HTML 应用的作用

- 提供垫上、塔架、Ladder Barrel、小器械、Wunda Chair 和 Reformer 动作库。
- 支持按分类筛选，也可以搜索中文或英文动作名称。
- 选择动作后填写每组重量、次数，并在女性肌肉示意图上查看主要参与肌群。
- 完成训练后生成训练摘要，并下载为 PNG 分享图。
- 当前版本为纯前端应用，不需要注册账号，也不需要后端服务器。

## 在线打开

本仓库已配置 GitHub Pages。GitHub Actions 完成部署后，可以通过下面的地址打开：

<https://sherrycw.github.io/pilates-record/>

如果地址暂时打不开，请在仓库的 **Actions** 页面确认 `Deploy Pilates library to GitHub Pages` 工作流已经成功运行；第一次部署通常需要等待一会儿。

### 手机打开

1. 用手机浏览器打开上面的 GitHub Pages 地址。
2. iPhone 可以在 Safari 中点击“分享”→“添加到主屏幕”。
3. Android 可以在 Chrome 中点击菜单→“添加到主屏幕”或“安装应用”。

添加到主屏幕后，它可以像普通手机应用一样从桌面打开；训练记录仍然只在当前浏览器页面中使用。

### 网页端打开

在电脑浏览器直接打开上面的地址即可。建议使用最新版 Chrome、Safari、Edge 或 Firefox。

## 本地运行

```bash
npm install
npm run dev
```

启动后打开 <http://localhost:5173/>。

## 构建与预览

```bash
npm run build
npm run preview
```

## 隐私承诺

我们承诺：**不保留用户数据，完全保护用户隐私。**

- 本项目不要求注册、登录或提交个人信息。
- 训练动作、重量、次数和组数不会上传到服务器，也不会写入本项目的数据库。
- 当前版本不会使用 `localStorage`、Cookie 或其他持久化存储保存训练记录；刷新或关闭页面后，当前页面中的训练数据会消失。
- 生成的 PNG 只在用户主动点击下载后保存到用户自己的设备；项目不会接收或保存这张图片。
- GitHub Pages 只负责提供网页静态文件，不负责收集或保存训练记录。

请注意：浏览器、操作系统和网络服务商可能有各自独立的日志或隐私政策；本项目不控制这些外部服务。隐私承诺针对本应用自身的数据处理行为。
